Below is a **point-wise explanation in easy language for exam preparation**. Each topic is explained **simply so you can remember and write in exams easily**.

# 1. Physical Database Design Issues

Physical database design means **how the database is stored on physical storage like hard disks**.

### Important Issues

1. **Storage Structure**
   How data will be stored on disk.

2. **File Organization**
   How records are arranged in files.

3. **Indexing**
   Creating indexes to speed up searching.

4. **Access Time**
   Reducing time needed to retrieve data.

5. **Storage Space**
   Using disk space efficiently.

6. **Performance**
   Ensuring faster query processing.

---

# 2. Storage of Database on Hard Disks

1. Databases are stored on **secondary storage devices like hard disks or SSDs**.
2. Data is stored in **files**.
3. Files are divided into **blocks or pages**.
4. Each block stores **multiple records**.
5. DBMS reads and writes data **block by block**.

**Hierarchy**

Disk → Blocks → Records → Fields

---

# 3. File Organization

File organization defines **how records are stored in a file**.

### Purpose

* Efficient data storage
* Faster retrieval
* Easy update and deletion

---

# 4. Types of File Organization

1. Heap File Organization
2. Sequential File Organization
3. Indexed File Organization
4. Hashed File Organization

---

# 5. Heap Files (Unordered Files)

1. Records are stored **in random order**.
2. New records are **added at the end of file**.
3. No sorting of records.

### Advantages

* Simple to implement
* Fast insertion

### Disadvantages

* Slow searching
* Need to scan entire file

Example
Student records stored without any order.

---

# 6. Sequential File Organization

1. Records are stored **in sorted order based on key**.
2. Usually sorted by **primary key**.

Example

Student records sorted by **Roll Number**.

### Advantages

* Fast sequential access
* Easy for batch processing

### Disadvantages

* Insertion and deletion are difficult
* Requires file reorganization

---

# 7. Indexed (Indexed Sequential) File Organization

1. Uses an **index to locate records quickly**.
2. Records are stored **sequentially**, but an **index table** is maintained.
3. Index stores **key value and record location**.

Example

Index

| Roll No | Address |
| ------- | ------- |
| 1       | Block A |
| 2       | Block B |

### Advantages

* Faster searching
* Efficient retrieval

### Disadvantages

* Extra storage required for index
* Index maintenance needed

---

# 8. Hashed File Organization

1. Uses **hash function to find record location**.
2. Hash function converts **key value into storage address**.

Example

Hash(Key) → Storage Location

Student ID → Hash Function → Disk Block

### Advantages

* Very fast data retrieval
* Efficient for exact searches

### Disadvantages

* Not suitable for range queries
* Hash collision may occur

---

# 9. Types of Indexes

Indexes help **speed up searching of records**.

### Main Types

1. Primary Index
2. Secondary Index
3. Clustered Index
4. Non-clustered Index

---

### Primary Index

1. Built on **primary key**.
2. Data is **sorted according to primary key**.

---

### Secondary Index

1. Built on **non-key attributes**.
2. Used when searching by **other attributes**.

Example

Search student by **Name**.

---

# 10. Index and Tree Structure

Indexes are often implemented using **tree structures**.

### Common Structure

**B-Tree / B+ Tree**

### Features

1. Balanced tree structure.
2. Fast searching, insertion, deletion.
3. Used in most modern DBMS.

Example

```
        50
      /    \
    20      80
```

Searching becomes **very fast**.

---

# 11. Multikey File Organization

1. File organization where **records can be accessed using multiple keys**.
2. Allows searching using **different attributes**.

Example

Student file can be searched by

* Roll Number
* Name
* Department

### Advantage

More flexible searching.

---

# 12. Need for Multiple Access Paths

Sometimes users need to **access records using different attributes**.

Example

Student table may be searched by:

* Roll Number
* Name
* Course

So database must provide **multiple access paths using indexes**.

### Benefits

* Faster searching
* Flexible queries
* Better performance

---

# 13. Multi-list File Organization

1. Records are connected using **multiple linked lists**.
2. Each list corresponds to **a different search key**.

Example

Student records linked by

* Department list
* Course list

### Advantage

Allows **fast access through different keys**.

---

# 14. Inverted File Organization

1. Uses **multiple indexes for different attributes**.
2. Each index points to **records that contain specific values**.

Example

Index for **Subject**

| Subject | Record List |
| ------- | ----------- |
| DBMS    | R1, R5, R8  |
| OS      | R2, R4      |

### Advantages

* Very fast searching
* Efficient for large databases

### Disadvantages

* Requires extra storage
* Index maintenance required
